## 你现在唯一需要谨慎的点

### Gate N 稳定但很低（~14%）意味着两种可能

* **可能 A（正常）：Gate N 量的是“分类纯度”，语义流主要改善的是“语境关联”，并不会显著提高 taxonomy 一致性**。此时 Gate N 作为“防崩塌指标”是合格的，但不应被当作“语义提升指标”。
* **可能 B（问题）：语义力虽然把 PMI 边拉近了，但没有转化为可读邻域（仍然局部混乱）**。这种情况在小样本上不一定暴露，需要全量边覆盖后再判断。

所以：**Gate N 保持稳定说明“可安全放量”，但不能证明“全量一定能把对话语义变强”。** 你还需要在全量跑里同时监控 Gate S/H（以及你们此前成熟的 Gate C/L/R/M）。

## 我建议的全量测试方案（最稳、最省回滚成本）

### Step 1：全量 PMI 构建（先构图，不先长训）

* 用 `scripts/build_wiki_pmi.py` 跑完整语料或你们设定的目标规模，输出：

  * 总边数、覆盖的词林叶子占比（coverage）
  * 每节点平均度、Top-K 截断后的度分布（P50/P90/P99）
  * 低频截断后保留的边比例

**退出条件**：如果覆盖率很低（例如大量叶子无 PMI 邻居），先补“定义句边/释义边”再继续训，不然效果会偏“少数词变强，多数词不动”。

### Step 2：短程全量训练（2k steps，验证“无副作用 + 方向正确”）

* Weight schedule：0 → 0.5 逐段退火（例如 0.1/0.2/0.3/0.5），每段 500 steps
* 每 250 或 500 steps 跑一次审计：

  * Gate C：Residual P99、Renorm Max、NaN
  * Gate L：纯中文保持
  * Gate H、Gate S
  * Gate N（防崩塌）
  * 你们的 PMI Dist Ratio（确认语义力没掉）

**退出条件（硬）**：

* Gate C 恶化 >2× 或出现 NaN
* Gate H 明显下滑（例如 <0.60 且持续）
* Renorm Max 突然抬升到你们历史安全线之上

### Step 3：长程全量训练（10k steps）仅在 Step 2 全绿后启动

* 目标就是你们写的：推动 Gate S < 0.90、Gate H > 0.80
* 但要接受一个事实：如果 H 主要由骨架决定，而语义流更偏“横向语境”，H 未必涨得那么快；必要时把目标拆成：

  * H：保持不下降 + 稍升
  * 新增 Gate（语境近邻可读性/同域命中率）作为语义主指标
