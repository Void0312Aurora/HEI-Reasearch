你们这份 1-Day Audit 报告整体上说明：**迁移到 Cilin 的方向是正确的，Base 已经从“混杂不可用”跃迁到“可交互、可读出、可扩展”的状态**。但按你们自己设定的“Base-Gold”标准来看，目前这份审计里有几项**证据链还不够闭环**，以及 **Gate L / Gate S / Gate C / Gate H / Gate M** 的表述需要更严谨，否则后续进入 Phase II/III 时容易出现“回归/解释不清”的问题。

下面逐 Gate 评估，并给出我建议的“最小补齐项”。

## 1) Gate L（Language Purity 96.80% vs 阈值 99%）

* **结论**：我同意“基本解决了语言混杂”，但**不同意把 96.8% 直接当作 PASS**，除非你把阈值改掉，或补充说明“99% 阈值不再适用的理由”和“剩余 3.2% 的来源已可控/可屏蔽”。
* **原因**：在 100k 的规模下，3.2% 的“非中文邻居”在交互 readout 里仍然可能以很高的频率冒头（尤其当某些 EN token 形成高密度团簇时）。

**建议你们做一个很小的补测**（几分钟就能做完）：

* 报告 **Purity 的分布**：P10/P50/P90（不是只有均值）。

  * 均值 96.8% 可能掩盖“部分概念仍严重混杂”的长尾。
* 把“非中文”拆分成：

  * 拉丁字母串 / 数字符号 / 缩写 / 拼音 / 其他
    这样才能决定是“词表残留”还是“训练机制问题”。

**最快的修复手段（不重训）**：

* 在 **Readout 端加 Language Gate**：中文交互时直接过滤非中文候选（几乎零成本，立刻把体感纯度拉到 ~100%）。
* 中期再做“词表清理/重训时屏蔽拉丁 token”，把 L 真正推到 ≥99%。

---

## 2) Gate S（Synonym Convergence Ratio 0.84）

* **结论**：方向是对的，但目前证据强度偏弱。0.84 的确意味着“同义组内距离 < 随机背景”，但需要明确：

  1. Ratio 的定义（是组内均值 / 随机均值？还是中位数？）
  2. 分布（是不是少数同义组很强、绝大多数接近 1？）

**建议补 2 个统计量**：

* 同义组 ratio 的 P50/P90（或均值±标准差）
* “显著性”不是必须做统计检验，但至少给出：

  * 有多少比例的同义组 ratio < 1（例如 78%）

如果你们想把几何邻域从“骨架可读”推进到“语义可读”，Gate S 最好从 0.84 进一步压到 0.7–0.8 区间（这通常要靠 Phase II 语义流边来做到）。

---

## 3) Gate C（Numerical Stability 仅 NaN=0）

* **结论**：NaN=0 是必要条件，但不是充分条件。你们之前已经有成熟的数值审计框架（Residual P99、Renorm Max、Constraint Violation 等）。迁移后 Gate C 只报 NaN，会让审计“退化”，不利于 Phase II/III。

**建议恢复至少两项**（与之前一致）：

* Residual P99
* Renorm Max（或 Constraint Violation steady）

这两项不难跑，且能证明“稳定来自物理求解，而不是靠投影兜底”。

---

## 4) Gate H（你用 R 代理）

* **结论**：不建议用 R 代理 H。Readout 覆盖率只能说明“树能展开”，不能证明“几何半径/距离与层级一致”。你们此前在 OpenHow 时代就见过“层级倒置”，因此 H 不能省。

**建议的最小 H 指标（非常容易实现）**：

* **Spearman(depth(code), radius)** 或 **Spearman(depth(code), mean_radius_of_words)**
* 或者更贴近词林：

  * 同一大类内的平均距离 < 跨大类平均距离
  * 同码前缀越长（更同类）距离越小（前缀长度 vs 距离的单调性）

只要补出这一个指标，你的“Base-Gold”证据链会更完整。

---

## 5) Gate M（Mapping “implied”）

* **结论**：不建议用 “implied”。因为你们接下来做交互，映射失败会直接变成“对话掉线”。
* **建议最小验收**：

  * 在一个固定词表（比如 top-50k 高频中文词 + 情感/意图专项清单）上，统计映射成功率
  * 明确报告：失败 Top-N（便于补词典/别名）

特别是你们已经提到过情感类，这里要把“爱、喜欢、开心、难过、害怕、想要、因为、如果”等做成专项。

---

## 6) Gate R（Avg 28 leaves/cat）

* **结论**：R 是正向信号，但建议你们补 **min / P10**。平均值可能很好，但若存在少量“叶子极少的死枝”，会导致某些输入输出很贫乏或重复。
